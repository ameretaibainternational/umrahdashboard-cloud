3. SYSTEM ARCHITECTURE
================================================================================

High-level flow
---------------

  [Browser]
      │
      ▼
  [Netlify — Next.js 16 App Router]
      │
      ├── Server Components → read data via lib/db.ts
      ├── Server Actions  → write data via app/actions/*.ts
      ├── API Routes      → app/api/storage/* (PDF download, presigned URLs)
      │
      ├──► Supabase PostgREST API (primary data path)
      │         └── PostgreSQL tables (bookings, payments, custom_invoices, …)
      │
      ├──► Cloudflare R2 (PDF blobs)
      │         └── Keys: invoices/{uuid}.pdf, vouchers/{uuid}.pdf
      │
      └──► (Optional) Direct Postgres via DATABASE_URL
                └── Disabled in production — use DISABLE_DIRECT_DB=true

Request auth flow
-----------------
  1. User logs in → Supabase Auth session stored in HTTP-only cookies
  2. proxy.ts (root middleware) checks session on every page load
  3. app/(app)/layout.tsx loads staff profile from staff_users table
  4. Server actions call requireAdmin() or requireModeratorFeature() before writes

Data layer pattern (lib/db.ts)
------------------------------
  - Demo mode  → in-memory demoStore (no Supabase needed)
  - Production → Supabase API, with optional direct Postgres fallback
  - PDF metadata in Supabase; PDF bytes in R2

Why DISABLE_DIRECT_DB is recommended
------------------------------------
  Direct Postgres (DATABASE_URL) can timeout from Netlify/serverless regions.
  When it fails, every request waits ~5 seconds before falling back to Supabase.
  Set DISABLE_DIRECT_DB=true (or remove DATABASE_URL) on Netlify for best speed.

Main tables
-----------
  staff_users       Staff profiles linked to auth.users
  company           Company name, phone, logo (single row)
  visa_settings     Visa SAR rates
  currency_settings SAR → PKR rate
  transport_rates   Bus/private transport SAR rates
  airlines          Ticket prices per airline
  hotels            Makkah/Madinah hotel room rates
  bookings          Saved packages from calculator
  payments          Customer payment records
  expenses          Supplier/expense records
  custom_invoices   Custom + package invoices (metadata)
  hotel_vouchers    Hotel voucher records
  invoice_settings  Default invoice footer/terms
  invoice_clients   Saved "Billed To" clients for dropdown
  storage_usage     Running total of PDF bytes (single row)

Row Level Security (RLS)
------------------------
  All tables use authenticated-only policies (auth_all pattern in migrations).
  Server actions enforce additional ownership checks for moderators.

Create .env.local for local dev. Mirror the same keys on Netlify.

Required — Supabase
-------------------
  NEXT_PUBLIC_SUPABASE_URL          https://{project-ref}.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY     Public anon key (safe for browser)
  SUPABASE_SERVICE_ROLE_KEY         Server-only — NEVER expose to client

Required — Cloudflare R2 (PDF storage)
--------------------------------------
  R2_ACCOUNT_ID
  R2_ACCESS_KEY_ID
  R2_SECRET_ACCESS_KEY
  R2_BUCKET_NAME                    e.g. umrah-dashboard-files

Recommended — Performance
---------------------------
  DISABLE_DIRECT_DB=true            Skip broken/slow direct Postgres; use Supabase API

Optional — Direct Postgres (usually NOT needed on Netlify)
----------------------------------------------------------
  DATABASE_URL                      Supabase pooler URI (port 6543)
  DATABASE_PASSWORD                 Alternative to embedding password in URL
  SUPABASE_DB_POOLER_HOST           Override pooler hostname if needed

Optional — Login bot protection (Cloudflare Turnstile)
------------------------------------------------------
  NEXT_PUBLIC_TURNSTILE_SITE_KEY
  TURNSTILE_SECRET_KEY
  Also whitelist your domain in Cloudflare Turnstile dashboard

Optional — Demo mode
--------------------
  NEXT_PUBLIC_SHOW_DEMO_CREDENTIALS=true   Show demo login hints on login page
  Leave NEXT_PUBLIC_SUPABASE_URL empty or containing "your-project" → demo mode
  (In-memory data only — resets on server restart)

12. FILE STORAGE (CLOUDFLARE R2)
================================================================================

Bucket purpose   : Store generated PDF files only
Metadata location: Supabase (custom_invoices, hotel_vouchers tables)
Usage tracking   : storage_usage table + Settings → Storage page

Storage admin tasks
-------------------
  Settings → Storage → "Sync usage & clear orphans"
  Removes R2 files with no active database reference and resets usage counter

When deleting invoices
--------------------
  - Delete from Invoices/Bookings page → also removes R2 PDF (if implemented)
  - Soft delete from Storage page → marks file_deleted_at, removes R2 blob
  - Hard delete without cleanup may leave orphan files — run sync orphans

R2 free tier
------------
  Cloudflare R2: 10 GB/month free storage
  App soft limit display: 9 GB (see lib/storage-config.ts)

13. KEY CODE PATTERNS (FOR DEVELOPERS)
================================================================================

Server actions (mutations)
--------------------------
  All write operations live in app/actions/
  Pattern: check permission → demo or Supabase/direct SQL → revalidatePath()

Reading data
------------
  Pages call functions in lib/db.ts (Server Components)
  Never query Supabase directly from client components for sensitive data

PDF upload flow
---------------
  1. Client generates PDF bytes (calculator / invoice forms)
  2. Base64 sent to server action
  3. app/actions/storage.ts → uploadInvoicePdfToStorage() → lib/r2.ts
  4. Metadata saved to custom_invoices or hotel_vouchers table

Auth middleware
---------------
  proxy.ts — redirects unauthenticated users to /login
  Uses cookie-based session (no network call on each request after cookie set)

Demo mode
---------
  lib/is-demo.ts + lib/demo-store.ts
  Activated when NEXT_PUBLIC_SUPABASE_URL is missing or placeholder

14. TROUBLESHOOTING GUIDE
================================================================================

Problem: App very slow (5–10 seconds per page/action)
Cause   : DATABASE_URL set but connection failing/timeouts
Fix     : Set DISABLE_DIRECT_DB=true on Netlify and in .env.local
          Remove or comment out DATABASE_URL
          Redeploy / restart dev server

Problem: "Could not save invoice: unexpected response from server"
Cause   : Server action timeout or PDF too large
Fix     : Ensure DISABLE_DIRECT_DB=true; retry usually works
          bodySizeLimit is 6mb in next.config.ts

Problem: Storage shows wrong usage (e.g. 0.7 MB but no files listed)
Cause   : Old storage_usage counter out of sync
Fix     : Settings → Storage → "Sync usage & clear orphans"
          Run 010_storage_usage_fix.sql in Supabase

Problem: Login redirect loop
Cause   : staff_users row missing for auth user
Fix     : Run staff link SQL (Section 10)

Problem: Turnstile not working on Netlify
Cause   : Missing env vars or domain not whitelisted in Cloudflare
Fix     : Add TURNSTILE keys to Netlify; whitelist ameretaiba.netlify.app

Problem: Accounts page crash / invoice edit crash
Cause   : Missing table or column in Supabase
Fix     : Run pending migrations; run setup-missing-tables.sql

Problem: Viewer can see buttons they shouldn't
Fix     : Check permission in component; viewer role hides save/delete in calculator

Problem: Storage dashboard shows 0%
Cause   : Was fixed to read from Supabase when direct DB unavailable
          Ensure latest code deployed
